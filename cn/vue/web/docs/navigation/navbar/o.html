工作原理

在开始使用导航栏之前，您需要了解以下内容：

  • 导航栏不需要添加 .navbar-expand {-sm | -md | -lg | -xl} 响应崩溃的类。的 .navbar-expand-lg 类是通过defoult添加的，但是您可以使用轻松地对其进行更改扩大支柱。您也可以使用配色方案道具或类。

  • 导航栏及其内容默认情况下是流畅的。采用可选的<mdb-container>限制其水平宽度。

  • 导航栏及其内容是通过flexbox构建的，可通过实用工具类和道具提供简单的对齐选项。

  • 导航栏默认情况下是响应式的，您可以轻松地对其进行修改以进行更改。

  • <mdb-navbar>组件默认情况下使用的是nav元素，但是您可以轻松地将其更改为更通用的元素，例如<div> ，只需添加一个 tag =“ div” 支撑到每个导航栏

请继续阅读以获取示例和支持的子组件列表。

支持内容

Navbars内置了对少数子组件的支持。根据需要从以下选项中进行选择：

  • <mdb-navbar-brand>您公司，产品或项目的名称。

  • <mdb-navbar-nav>全高和轻量级导航（包括对下拉菜单的支持）。

  • 用于任何形式的控件和动作。

  • <mdb-navbar-toggler>用于通过父断点分组和隐藏导航栏内容。

  • .navbar-text 用于添加垂直居中的文本字符串。
