弹窗

引导弹出窗口

这是用于将Bootstrap弹出窗口（如在iOS中找到的）添加到网站上任何元素的文档和示例。

使用popover插件时要了解的事项：

  • 弹窗依赖第三方图书馆 Popper.js 用于定位。您必须包括 popper.min.js 在bootstrap.js之前，以便弹出窗口起作用！
  • 出于性能原因选择启用弹出式窗口，因此你必须自己初始化它们 。
  • 零长度标题和内容值永远不会显示弹出框。
  • 指定容器：“身体” 以避免在更复杂的组件（例如我们的输入组，按钮组等）中呈现问题。
  • 在隐藏元素上触发弹出窗口将不起作用。
  • 的弹出式窗口 .disabled 要么残障人士元素必须在包装元素上触发。
  • 从跨越多行的超链接触发时，弹出窗口将居中。采用空白：nowrap; 在你的<a>避免这种行为。
  • 在从DOM中删除其对应元素之前，必须先隐藏弹出窗口。

继续阅读以了解一些示例如何处理弹出窗口。

禁用元素

与元素残障人士属性不是交互式的，这意味着用户无法将鼠标悬停或单击它们来触发弹出窗口（或工具提示）。解决方法是，您需要从包装器触发弹出窗口<div>要么<span>并覆盖指针事件在禁用元素上。

对于禁用的弹出触发器，您可能还希望 data-trigger =“ hover” 这样，弹出式窗口就可以立即显示为用户的视觉反馈，因为他们可能不会期望点击在禁用的元素上。

方法

异步方法和转换

所有API方法都是异步然后开始过渡 。转换开始后，他们会立即返回给呼叫者，但在结束之前 。此外，在过渡组件将被忽略 。

$（）。popover（选项）

将弹出处理程序附加到元素集合。

.popover（&#39;显示&#39;）

显示元素的弹出框。它会在实际显示弹出框之前返回给调用者 （即 shown.bs.popover 事件发生）。这被认为是弹出窗口的“手动”触发。标题长度为零的弹出窗口永远不会显示。

> $（&#39;＃element&#39;）。popover（&#39;show&#39;）;

.popover（&#39;隐藏&#39;）

隐藏元素的弹出框。在弹出窗口实际被隐藏之前，它将返回给调用者 （即 hidden.bs.popover 事件发生）。这被认为是弹出窗口的“手动”触发。

$（&#39;＃element&#39;）。popover（&#39;hide&#39;）;

.popover（&#39;toggle&#39;）

切换元素的弹出框。它会在实际显示或隐藏弹出窗口之前返回给调用者 （即 shown.bs.popover 要么 hidden.bs.popover 事件发生）。这被认为是弹出窗口的“手动”触发。

> $（&#39;＃element&#39;）。popover（&#39;toggle&#39;）;

.popover（&#39;dispose&#39;）

隐藏和销毁元素的弹出框。使用委派的弹出式窗口（使用的选择器选项 ）无法在后代触发元素上单独销毁。

$（&#39;＃element&#39;）。popover（&#39;dispose&#39;）;

.popover（&#39;启用&#39;）

使元素的弹出框具有显示能力。默认情况下启用弹出窗口 。

$（&#39;＃element&#39;）。popover（&#39;enable&#39;）;

.popover（&#39;禁用&#39;）

删除显示元素弹出框的功能。仅当重新启用弹出窗口时，才能显示该弹出窗口。

$（&#39;＃element&#39;）。popover（&#39;disable&#39;）;

.popover（&#39;toggleEnabled&#39;）

切换显示或隐藏元素的弹出框的功能。

$（&#39;＃element&#39;）。popover（&#39;toggleEnabled&#39;）;
